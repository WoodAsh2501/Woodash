忙到无脑……

最近的事情实在是非常的多，我甚至没什么空闲来读书、想事情： 我只有洗澡和失眠的时候才有空思考。

那天失眠的时候，我突然有点想读许立志，于是读到了五点钟。然后我突然有点难过，我的时间都被简陋的课程、盲目的作业和任务占据了，剩下给我的时间寥寥无几，我就不能思考一些真正重要的事情：设计、人生、社会……我真的很讨厌这样的日子。我觉得，这种课程甚至是一种副作用吧。来看这周的周报吧。

摘录

Altair 撞开了机房的大门，差不多在同一时间，MITS 公司的竞争对手在全美各地的车库里纷纷涌现。爱德华·罗伯茨的计算机价格很难再降，要不是 Altair 的交货时间总要拖延很久，MITS 公司的先发优势肯定是异常明显的。但是，这些出于爱好而入行的企业家，没有一个人的目标是挣大钱，结果大多数人以失败告终。尽管如此，在失败时，他们都是把资料和图纸公开后坦然离场的。前事不忘，后事之师。先行者的失败并没有使后来者气馁，后来者也没有放弃革新。这场革命的爆发源自内在的动力，而不是受外部利益的驱使。正因如此，微型计算机产业也没有遵循传统的经济规律。

好的软件开发原则

这篇文章介绍了作者总结的十条软件开发建议。感觉还是有些道理的，我简单 （再） 总结一下，不过还是建议去读原文好了。

  1. 让每一个 git 提交足够小。
  2. 让每次的修改都可以在十分钟内完成，这样可以帮助持续重构。
  3. 所有代码都是负担。
  4. 不要测试代码框架，因为有很多人已经测试过了。
  5. 如果一个函数没有足够合适的模块，就创建一个新的模块。
  6. 如果不知道一个 API 应该是什么样的，就先写一些测试，用客户的角度来思考。
  7. 复制粘贴一次是可以的，再重复就应该抽象出来了。
  8. 如果设计过时了，就改变它。不要太珍惜和难过。
  9. 技术债分为三种：阻止你现在做事情、阻止你以后做事情、可能阻止你以后做事情。
  10. 不容易测试的东西意味着设计需要改变。

我好爱喝醉之后听黑金属

我越来越喜欢嘈杂的音乐、声墙与轰然包裹着的噪声。在酒精的作用下，有概率获得更好一些的体验。

伏特加已经被我喝完了，剩下的朗姆酒冻在阳台，冻得 冻手 。我大概也就是随便勾兑着喝，乌龙茶、酸梅汤——我发现茉莉蜜茶其实加上伏特加味道挺好的。那天我只剩下一瓶维 C 了，而酒也只剩小半瓶。于是我决定来一次盛大的狂欢，去超市买了热狗、怪味豆，把剩下的酒全部喝掉了。

我感觉我以前没喝这么多过，以至于真的有点晕眩。喝多之后，我准备随便听点歌，于是随机到了一首黑金属。感觉真的很好，我觉得我喜欢这种感觉。

写完之后，我才想起那天我喝的其实是精酿啤酒。喝完朗姆酒的第二天我感觉有点头痛，觉得自己应该戒酒了。

游离是我的一部分

高三的时候，我发现我没办法集中注意力，甚至写作文都很难保持话题的统一——三年之后，我仍然在和自己挣扎。上学期认知心理学做的注意力分配实验，我的得分甚至是不及格的（虽说这粗略的实验不一定能说明什么）。我总是懊悔为什么我不能再专注一些，再做更多的事情。然后我突然想到，我或许应该接受自己的这一部分。

我应该接受自己游离的注意力，这也不一定是件坏事，或者说，不一定需要 矫正 。我需要的是在适应自己的情况下提升效率——比如我或许应该调整工作的节奏，一定时间后就切换一下任务之类的，因为我发现，如果让我一直做一件事情的话我的效率其实更低。

不过，传统的工作模式对我这种做法可能并不算特别友好……我还要再适应一下吧。

命令行狂热

自从这学期开始用 Linux 以来，我越来越喜欢命令行界面了。不觉得很酷吗，很符合我对嗨客的想象……

命令行快而高效，但代价是我们要形成对功能架构的记忆。我不 （像计算机老登一样） 觉得命令行会比图形用户界面更加「高级」，大家都是界面，无非是基于文字还是基于图形罢了。命令行界面用人类的认知负荷，来换取了操作效率，这是否值得，是见仁见智的——不过，图形用户界面的成功并非没有原因。

但从我深入接触命令行以来，我的确开始喜欢上这种界面了。或许其中很大一部分是（虚假的）成就感：我竟然可以用文字来让计算机做到我想做的事情！而完成相同的操作在现代的图形界面上就不会激起什么波澜，因为它们实在是 太直观易用 了。

同时，我觉得敲命令这件事情还是挺有反馈感的，你输入几个字符，然后按下回车，屏幕上就会给你长长短短的输出。我很喜欢看电脑输出的结果，尽管大多数时候都是我看不懂的日志，但仅仅是盯着屏幕上滚动的文本，就可以给我带来舒适的感觉。

我有时候会把命令行窗口调成全屏，然后放松下来，只是盯着屏幕上瀑布一般的输出，什么都不做……
